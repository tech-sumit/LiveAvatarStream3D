# MuseTalk (TMElyralab/MuseTalk) pins diffusers/transformers/numpy versions and
# needs the OpenMMLab dwpose stack (mmcv/mmdet/mmpose) for landmark detection
# during avatar preparation. None of these are compatible with the realtime
# service's system interpreter (coqui-tts/transformers 4.57, torch 2.4 — which
# has no prebuilt mmcv wheels). So MuseTalk runs in its own venv driven by

#
# This venv is built WITHOUT --system-site-packages (it ships its own torch 2.1
# + cu121, the newest combo with prebuilt mmcv wheels and good H100/sm_90
# support). Installed by deploy/setup_musetalk.sh; this file is the manifest.
