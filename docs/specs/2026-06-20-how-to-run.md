# How to run LiveAvatarStream3D

**Date:** 2026-06-20

## Prerequisites

- Node.js ≥ 20
- `npm install` from repo root (workspaces)
- For GPU work: RunPod account, `RUNPOD_API_KEY` in project `.env`
- For deploy: Cloudflare Wrangler, D1/R2/queue already provisioned

---

## Scene editor (local)

The editor is the upstream **three.js editor** with a **Render** sidebar tab for LAS.

```bash
cd projects/LiveAvatarStream3D
npm install
npm run dev:editor
```

- **URL:** http://localhost:5174
- **API:** `apps/scene-editor/.env.development`

```bash
VITE_API_URL=https://las-control-api.tech-sumit.workers.dev/api
```

Use the **deployed Worker**, not local `wrangler dev`, so uploads land in production R2 and the `las-jobs` queue reaches the GPU pod.

### Editor workflow

1. Open **Render** tab (default)
2. Select a **ready** cloned voice (deduped dropdown)
3. Edit the **single script line**
4. Use **Scene** tab — orbit camera, import GLB, adjust lights
5. Click **Record (engine render)** → job id + status; download MP4 when `succeeded`

### Build production bundle

```bash
npm run build --workspace apps/scene-editor
# output: apps/scene-editor/dist/
```

## Control API (deploy)

```bash
cd services/control-api
wrangler secret put GPU_PROVIDER_TOKEN
wrangler secret put INTERNAL_SERVICE_TOKEN
wrangler deploy
```

Confirm `GPU_PROVIDER_BASE_URL` in `wrangler.toml` matches the live pod gateway.

**Pending deploy (local only):** `DELETE /api/voices/:id` for duplicate POC voice cleanup in the editor.

## H100 pod lifecycle

### Inspect / provision

```bash
./scripts/gpu/spawn-pod.sh --info
# POD_ID, GATEWAY_URL, SSH, cost/hr
```

### Stop / start (save GPU $)

```bash
# Stop — volume persists, container disk wiped on next start
curl -X POST -H "Authorization: Bearer $RUNPOD_API_KEY" \
  "https://rest.runpod.io/v1/pods/${POD_ID}/stop"

# Start
curl -X POST -H "Authorization: Bearer $RUNPOD_API_KEY" \
  "https://rest.runpod.io/v1/pods/${POD_ID}/start"
```

After **start**, re-run pod bring-up (`POD_SETUP.md`): `install_deps.sh` at minimum.

### Resume helper

```bash
./scripts/gpu/resume-pod.sh --print
```

## Sync engine-three to pod (WYSIWYG fix)

Required when local `engine-three` has WYSIWYG / Lee Perry-Smith changes not on the pod.

```bash
POD_SSH=root@213.181.105.227 \
POD_SSH_PORT=11422 \
LAS_SSH_KEY=~/.ssh/las_runpod \
./scripts/gpu/sync-engine-three.sh
```

Verify:

```bash
curl -s https://s5jwghwjkwo96q-8080.proxy.runpod.net/engine-three/health | python3 -m json.tool
# expect: wysiwygScene: true, leePerrySmithLoaded: true
```

If `supervisorctl restart engine-three` fails, SSH in and run `start.sh` from `services/gpu/deploy/`.

## Troubleshooting

| Symptom | Likely cause | Fix |
|---|---|---|
| Voice clone stuck `cloning` | Old waitUntil path or pod down | Queue consumer + pod health; retry via API |
| Record 400 empty script | Validation | Fill script line in Render tab |
| Record 500 | Invalid spec | Check Worker logs; ensure voice `ready` |
| Manifest "not ready" during `tts` | Normal — manifest written in `compiling` | Poll after TTS completes; use **full** job id |
| MP4 shows stick figure, not Lee bust | Stale pod `engine-three` | Run `sync-engine-three.sh`, verify health |
| Camera wrong in MP4 but manifest correct | Same — legacy render path on pod | Sync + restart engine-three |
| DELETE voice fails | Endpoint not deployed | `wrangler deploy` control-api |
